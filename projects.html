Analytics Technology Map

28.04.2020

Github Repository of project
Description

The goal of this project is to create a map of technologies that are related to Data Analytics. This includes database technolgies, data processing technologies and data visualization tools. Technologies should appear close to each other on the map if they are either similar (HBase, Cassandra) or if they are often used together (HBase, Phoenix). I originally got the idea when working at a consultancy where we programmed an Artificial Intelligence that would make suggestions on which consultant should be staffed on a given project. In order to generate suggestions, we mapped all available consultants in a high dimensional vectorspace using word2vec. We then mapped each project in the same vectorspace and calculated the consultants that were closest. (This is a simplified version of how the final product worked). Here is a basic description of how these vectors can be generated talk of David Kriesel, in which he described his works related to the analysis of the articles of a large german newspaper. His talk is on German, but there is an English version and I highly recommend watching it, since it is very interesting to see not only his visualizations and results, but also what is possible to derive from metadata.
German version, English version
In the visualization that serves as an example for what I am trying to do, David Kriesesl maps the Keywords of news articles (policics, economy, Angela Merkel) based on how often they appear as keywords together.

Initial Approach

Data

As a data source, I need a large amount of text in which the application of technologies that are interesting to us are described. Ideally, the data would consist of As a data scource, I am going to use posts and comments from reddit.com.
Reddit.com is a network of communities based on people's interests. There are several subreddits, which make up subcommunities in which people share content and ask questions. Other members of a subreddit can see and comment on the posts.

Analytics

Since the raw data that I am going to use is not structured in any way, I am going to extract features in a very simple way. I am going to write a json file with technologies that I want to include in my analysis. For each technology, I am going to specify aliases (Spark = Spark2, vue = vue.js, S3 = Simple Storage Service) and the corresponding technology family (Spark: Analytics, MongoDB: Database). Use a structured list of technologies that should be considered structured in a json file. This lets me use aliases . Then look through each post and through each comment and find out if the technologies that I have specified apppeared in the post/comment. This has one major drawback: Some technolgies that I would like to include have names that correspond to English words, such as the programming language Go and the Javascript framework React. A possible way around would be to have a Natural Language Processing model analyze which of the words or group of words in the comments or posts correspond to a named entity (Basically, which of the words are names). I will experiment with ways to deal with the drawback later, and will ignore "problematic" technolgies in the first version of the technology map.

Testing and Visualization

In general, Natural Language Processing models use a vectorspace for the embedding of words or phrases that is high-dimensional, typically in the hundreds. We do not need to map the large number of words, that languages normally use, into our vectorspace. Therefore, our vectorspace will be much smaller in terms of its dimensionality. In my past experience, the optimal number of dimensions for a vectorspace that had to contain 1000-2000 different vectors was in the range of 10-20. If we want to visualize the entire map, we would have to find a way to plot all technologies in two dimensions. If we decide to just use a two dimensional map, we can use Gephi, the tool that David Kriesel used a tool for his visualization. Fortunately, he gives a description of how he used the tool on his website. There is still a lot of work ...given... for using the tool.
Use basic visualizations to test if our model is working. For example, test which are the technologies closest to a given test technology and visualize the distance of each of the resulting technologies to our test technology (maybe just use a basic table). If results are good, see how low we can go in terms of the vectorspaces dimensionality. If d=2 still gives good enough, try out Gephi.

Current status

I used the pushlift API to acess old reddit data. I chose to include posts and comments from subreddits "MachineLearning", "webdev", "bigdata", "datascience", "analytics" and "ArtificialInteligence". I decided to use data from the last 2 years, to get a somewhat recent view on things. I also decided only to include posts and comments that have as least as many likes as dislikes. This might be overcautious, since most of the time the comment was probably not downvoted because the technologies named should not appear together. Under the given requirements (subreddit, time, vote) I get more than 70.000 posts and more than 900.000 comments with a content that contain more than 15 characters. There are two ways how the data can be used:
1: Treat every post and every comment individually. This would mean that I count two technologies as appearing together only if they appear in the same post/comment. 2: Combine contents of a post and its comments. Either "globally", by just combining all of their contents or by building several "local" combinations, in which each comment would be combined only with its father comment (which could be the original post). I will start by first analyzing each post and each comment individually, since it appears more straightforward. A post or content has an effect on the outcome of the model, if at least two different technologies appear in it.
